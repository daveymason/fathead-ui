Frequently Asked Questions

Everything you need to know about Fathead UI

Fathead UI systematically applies the golden ratio (1.618) to all spacing and sizing. Our CSS variables follow a precise mathematical progression: --fh-xs is the base divided by 1.618, while each larger size multiplies the previous by 1.618. This creates naturally harmonious proportions in margins, padding, font sizes, and component dimensions.

Fathead UI is built on core Human-Computer Interaction principles including visibility of system status, user control and freedom, consistency, error prevention, recognition over recall, flexibility, and aesthetic minimalism. Our components focus on providing clear feedback, intuitive interaction patterns, and reducing cognitive load for users.

Fathead UI prioritizes accessibility with WCAG 2.1 AA compliant color contrast ratios, semantic HTML structure, proper ARIA attributes, keyboard navigation support, and touch targets meeting minimum size requirements (44px). We also support reduced motion preferences and provide focus indicators for all interactive elements to ensure a truly inclusive experience.

Fathead UI achieves interactivity primarily through CSS techniques like the checkbox hack, CSS variables, and modern selectors rather than relying on JavaScript. This approach results in faster load times, better performance, reduced dependencies, and improved reliability. Even complex components like accordions, modals, and tooltips are implemented with pure CSS.

Our color system is based on mathematical relationships between complementary colors. The primary blue (#219EFF) and complementary orange (#FF8A00) form the foundation, with secondary colors derived using golden ratio proportions. This creates harmonious color schemes that maintain appropriate contrast ratios while establishing visual hierarchy and emotional resonance.
