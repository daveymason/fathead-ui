Getting Started

To use Fathead UI, add this stylesheet link to your project:

Add to your project:

<link rel="stylesheet" href="https://daveymason.github.io/fathead-ui/fathead-ui.css" />

Simple Installation

Just add the stylesheet link to the head of your HTML document and you're ready to start using Fathead UI components.

Wrap your app in <body class="fh-ui" data-theme="light"> for theme support.

Components

  • Navbar: .fh-ui-navbar – Responsive, sticky, glassmorphic navigation bar.
  • Button: .fh-ui-button – Gradient, animated, accessible buttons.
  • Card: .fh-ui-card – Glassmorphism, shadow, and hover effects.
  • Sidebar: .fh-ui-sidebar – Vertical navigation, glassmorphic, responsive.
  • Hero Section: .hero-section – Large, gradient, animated intro section.
  • Code Snippet: .code-snippet – Highlighted code with copy button.
  • Floating Link: .floating-link – Fixed, glassy, animated call-to-action.
  • Alert/Feedback: .fh-ui-feedback – Accessible, animated info banners.

Utility Classes

  • .fh-ui-glass – Apply glassmorphism to any element.
  • .fh-ui-holo-icon – Holographic animated icon effect.

Golden Ratio & Spacing

Fathead UI uses the golden ratio (1.618) for all spacing and sizing. Core variables:

  • --fh-base: 1rem (base unit)
  • --fh-xs: calc(var(--fh-base) / 1.618)
  • --fh-sm: calc(var(--fh-xs) * 1.618)
  • --fh-md: var(--fh-base)
  • --fh-lg: calc(var(--fh-base) * 1.618)
  • --fh-xl: calc(var(--fh-lg) * 1.618)
  • --fh-xxl: calc(var(--fh-xl) * 1.618)

Use these variables for margin, padding, gap, and font-size for harmonious layouts.

HCI & Accessibility

  • All components use high-contrast colors and large touch targets.
  • Keyboard navigation and focus states are built-in.
  • WCAG 2.1 AA color contrast and ARIA best practices.
  • Animations respect prefers-reduced-motion for accessibility.

Customization

Override any :root CSS variable to change theme colors, spacing, or fonts. Example:

:root {
  --fh-primary: #2DC0FF;
  --fh-accent: #FF8A00;
}

Switch between light and dark themes by changing data-theme on <body>.

CSS-Only Implementation

Fathead UI uses pure CSS solutions with no JavaScript dependencies, making it lightweight and compatible with any JavaScript framework (or none at all).

Interactive Components

  • Copy Buttons: Use CSS-only animations for feedback when clicked. The ::before pseudo-element slides into view on click and fades out using keyframe animations.
  • Popups & Dialogs: Implemented using hidden checkboxes for state management. When a <label for="checkbox-id"> is clicked, the associated checkbox toggles and CSS shows/hides the popup using the :checked selector.
  • Toasts: Use checkbox-based state with auto-hiding animations. The toast appears when the checkbox is checked and automatically fades out after a set duration using CSS animations.
  • Loaders: Show and auto-hide with CSS animations. The loader-auto-hide animation automatically hides the loader after a delay when triggered.
  • Active Navigation: Sidebar links use the .active class for highlighting, which is applied based on the current section using only CSS.

Example: CSS-Only Modal

<!-- Hidden checkbox for state management -->
<input type="checkbox" id="my-popup-checkbox" class="fh-ui-popup-checkbox">

<!-- Trigger label styled as a button -->
<label for="my-popup-checkbox" class="fh-ui-button">Open Popup</label>

<!-- The modal/popup -->
<div class="fh-ui-popup" id="my-popup">
  <div class="fh-ui-popup-content">
    <p>Popup content</p>
    <label for="my-popup-checkbox" class="fh-ui-button">Close</label>
  </div>
  <label for="my-popup-checkbox" class="fh-ui-close-overlay"></label>
</div>
